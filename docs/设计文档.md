## **基于ELF 2 (RK3588) 的虚实融合智能植物养护系统**

**摘要**

本作品以飞凌ELF 2开发板（瑞芯微RK3588处理器）为核心平台，设计了一套融合环境感知、边缘AI推理、大模型对话与虚拟角色交互的智能植物养护系统。系统围绕"给植物一个灵魂"的核心理念，构建了从物理温室到虚拟角色的完整映射链路：ELF 2通过UDP协议接入多路环境传感器（温度、湿度、CO₂、光照、气体、土壤），运行植物健康分析引擎将数据转化为可量化的健康洞察；利用RK3588内置6TOPS NPU部署YOLOv5模型进行端侧植物生长视觉检测；在本地运行DeepSeek驱动的AI角色"酢酱酱"——一株被赋予生命的拟人化酢浆草——其情绪和表达由实时传感器数据驱动，实现了"土壤干了她会说渴、阳光太强她会喊热"的情感映射机制。系统通过MQTT协议与EMQX Cloud连接，支持远程查询与控制。PC端运行VTube Studio Live2D虚拟形象与讯飞TTS语音合成，作为角色的"面孔与声音"，通过MQTT与ELF 2上的AI大脑协同工作。K230微控制器作为感知执行节点驱动7路传感器与5路执行器。

整套系统体现了"虚实结合"的设计哲学：物理世界的传感器数据驱动虚拟角色的情绪变化，而用户与虚拟角色的对话又可以触发对物理设备的控制——浇水的请求由酢酱酱"转达"给水泵，补光的建议从她的口中说出。这种将嵌入式感知、边缘AI与虚拟角色深度融合的方案，为智慧农业的人机交互提供了全新范式。

**第一部分 作品概述**

**1. 功能与特性**

本系统围绕"让植物拥有一个数字灵魂"的设计目标，构建了以下功能模块：

（1）**多源环境感知与健康评估**：通过7路传感器以1Hz频率采集温室环境数据（温湿度SHT35、CO₂/TVOC CCS811、光照BH1750、气体Mics5524、土壤湿度HW390），经UDP协议汇聚至ELF 2。在ELF 2本地运行植物健康分析引擎：四维健康指数评分（温度/湿度/光照/土壤各贡献0-25分）、300秒滑动窗口趋势检测、24小时自适应基线异常检测。所有传感器数据同步写入CSV日志与MySQL数据库。

（2）**智能控制与决策**：采用双层决策架构。底层6条本地优先级规则（P0安全保护→P1高温散热/干旱浇水→P2补光加湿→P3停水保护）保证毫秒级确定性响应；上层在满足事件触发条件时调用DeepSeek API获取智能建议。决策经硬安全上限过滤后通过UDP指令下发至执行层。

（3）**端侧AI视觉检测**：利用RK3588 NPU运行YOLOv5模型，对温室视频流（1280×720@90fps H.265）进行植物新叶与新花的生长识别。检测结果经连续确认与冷却去重后，通过MQTT推送通知。

（4）**虚实融合的情感映射引擎**：系统的核心创新——将物理传感器数据映射为虚拟角色的情绪状态。温度过高→角色表现出"热"的不适感；土壤偏干→角色表达"口渴"；光照不足→角色显得"蔫蔫的"；检测到新叶→角色表现出惊喜与开心。这种映射不是简单的阈值规则，而是经过LLM语义理解后生成的自然语言表达和对应的情绪标签，再驱动Live2D虚拟形象的面部表情与身体动作。

（5）**AI角色智能对话**：在ELF 2上运行完整的LangGraph Agent工作流——长期记忆加载→RAG植物知识库检索→Bing联网搜索→角色化回答生成。酢酱酱以"我是一株酢浆草"的第一人称视角与用户交流，专业知识用亲身体验的口吻讲述，陌生话题用植物视角趣味回应。

（6）**多平台交互接入**：PC端作为角色的"面孔"，通过MQTT与ELF 2上的AI大脑通信，支持B站直播弹幕、QQ群聊/私聊、微信三平台的并发消息处理，配合VTube Studio实时驱动Live2D虚拟形象的口型、表情和动作。

**2. 应用领域**

- **智慧农业交互革新**：突破传统传感器仪表盘的交互模式，用户通过与虚拟角色的自然对话即可了解植物状态——"酢酱酱你渴不渴？"比查看土壤湿度读数更直观、更有温度。适用于温室大棚、植物工厂的日常管理。
- **家庭园艺助手**：ELF 2的低功耗与紧凑尺寸适合家庭部署。AI角色让植物养护变得有趣——她会主动提醒浇水、分享养护知识、对主人的照顾表达感谢。
- **科普教育与展示**：虚拟角色+真实传感器的模式极具展示性。参观者可以与"活的植物"对话，直观感受环境因素如何影响植物生长。已在B站直播中验证了良好的互动效果。
- **嵌入式AI教学**：系统完整覆盖了感知采集→数据分析→AI推理→虚拟角色驱动→云端通信的技术链条，展现了ELF 2作为边缘AI平台的综合能力。

**3. 主要技术特点**

（1）**虚实融合的情感映射机制**：系统的标志性创新。建立了从物理传感器到虚拟角色情绪的映射管线——传感器数值→分析引擎（健康评分/趋势/基线）→LLM语义转换→情绪标签（8种）→VTube Studio表情参数。酢酱酱不是简单地"播报数据"，而是像一个真正的生命体一样"感受"环境：光照增强时她的叶子（Live2D参数）会舒展开来，土壤变干时她会微微蹙眉，检测到新叶时她会惊喜地睁大眼睛。这种设计让用户产生情感共鸣，将冷冰冰的传感器数据转化为有温度的交互体验。

（2）**云端-边缘协同架构**：ELF 2作为边缘计算节点承担全部实时性要求高的任务——传感器处理、本地规则控制（<50ms响应）、NPU推理（<100ms）。云端服务（DeepSeek API、EMQX Cloud）承担非实时的大模型推理与消息路由。本地规则可脱离云端独立运行，LLM失败时自动降级为纯本地决策。这种协同兼顾了嵌入式系统的实时可靠性与云服务的智能灵活性。

（3）**多源数据融合驱动的角色智能**：酢酱酱的对话不是通用聊天AI的照搬，而是由四类数据共同塑造——传感器实时数据（我知道现在有多热）、历史记忆（我记得你上周问过浇水频率）、植物知识库（酢浆草原产南非所以喜欢阳光）、联网信息（今天天气预报说要下雨）。四者在LangGraph工作流中有序融合，使角色的每次回应都有"根"。

（4）**RK3588 NPU的端侧AI深度利用**：YOLOv5植物生长检测模型部署在NPU上，H.265视频解码→OpenCV预处理→NPU推理→结果去重→事件通知的完整流水线在ELF 2本地内存中完成。6TOPS算力使10秒间隔的推理任务在毫秒级完成，其余时间NPU处于空闲，为未来部署更多模型（病害检测、成熟度判断）预留了充足的算力。

（5）**高鲁棒性的系统设计**：各子系统独立运行、故障隔离。视频流断开不影响传感器采集与控制；MQTT断连自动重连；LLM API超时/失败自动降级为本地规则，连续失败3次暂停5分钟；串口断连指数退避重连。每个I/O通道的异常都不会波及其他模块。

**5. 主要创新点**

- **创新点一：虚实融合的情感映射机制**。本系统首次将嵌入式传感器数据流与Live2D虚拟角色的情绪系统建立了深度映射关系。不同于简单的"温度高了报警"，酢酱酱用拟人化的方式"感受"环境——传感器的每一次读数变化都可能触发她细微的表情波动。这种映射包含三层：①数值层（传感器→健康评分→趋势判断）、②语义层（LLM将数据转化为角色化的自然语言表达）、③表现层（情绪标签→VTube Studio Live2D参数→面部表情+身体动作）。用户看到的不是一个仪表盘，而是一个会因为土壤干了而蹙眉、因为新叶萌发而惊喜的"活着的植物"。

- **创新点二：传感器数据驱动的大模型角色人格**。传统AI角色依赖预设人设文本和对话历史，而酢酱酱的"人格"有物理根基——她的喜怒哀乐与真实温室环境状态相连。当用户问"你今天感觉怎么样？"，她的回答不是随机的，而是基于当前温度、湿度、光照的综合感受。这种将物理世界实时数据注入大模型对话上下文的做法，让AI角色的行为具有了"真实性"和"可验证性"，开创了物联网场景下大模型应用的新范式。

- **创新点三：云端-边缘协同的双层决策架构**。在ELF 2上实现了一种兼顾实时性与智能性的双层控制架构——本地规则保证安全保护的确定性毫秒响应（无需网络），LLM智能顾问在满足统计学触发条件时提供更精细的调节建议。两者通过硬安全上限和自动降级机制协同工作。这种架构让嵌入式系统既能独立可靠运行，又能借助云端大模型获得超越预设规则库的智能。

- **创新点四：以角色为界面的智慧农业交互范式**。用户不需要学习查看传感器仪表盘、理解专业术语——他们只需要和一个可爱的酢浆草角色聊天。"你渴了吗？"比"土壤湿度ADC值是否低于阈值"更自然；"哇你长新叶子了！"比YOLO检测告警更有温度。这种交互范式降低了智慧农业的使用门槛，让植物养护从"技术活"变成了"交朋友"。

**6. 设计流程**

系统设计遵循"从物理感知到数字灵魂"的递进路径。第一阶段：完成传感器驱动与执行器控制的底层验证（I2C/ADC/PWM逐一调试），确保物理层数据通路畅通。第二阶段：在ELF 2上搭建数据接入与分析框架——UDP传感器接收→CSV/MySQL存储→健康评分→趋势检测→本地规则控制。第三阶段：引入AI能力——DeepSeek LLM对话→YOLOv5 NPU视觉检测→MQTT云端通信。第四阶段：构建虚实融合的情感映射管线——传感器→LLM语义转换→情绪标签→VTube Studio表情驱动，赋予角色"感受"环境的能力。第五阶段：多平台交互接入（B站/QQ/微信）。全程以ELF 2为开发与测试的主平台。

**8. 硬件系统介绍**

**9. 软件系统介绍**

**9.1 整体架构**

ELF 2上运行两套协作的Python子系统（总计约6,000行）：

- **子系统A — 温室智能控制器（sensor_listener/）**：asyncio主循环+视频线程+MQTT线程。负责UDP传感器接入、健康分析引擎、双层决策控制、H.265视频接收与YOLOv5 NPU推理、MQTT指令路由、CSV/MySQL存储。

- **子系统B — AI角色引擎（elf2/）**：6协程asyncio并行架构。负责UART语音输入、意图路由、LangGraph Agent工作流（记忆→RAG→搜索→回答）、情感映射、Piper TTS语音合成、VTube Studio表情驱动。

K230端运行MicroPython 4线程架构（传感器采集、命令接收、执行器PWM+LED动画、视频推流）。PC端作为角色的"面孔"运行VTube Studio Live2D渲染与讯飞TTS语音合成。

**9.2 核心软件模块**

**模块1：UDP传感器接入与健康分析（protocol.py + analysis.py）**

ELF 2以asyncio DatagramProtocol监听UDP:8259端口。每个JSON数据包经解析后驱动以下流水线：丢包检测→健康评分计算（四维融合，0-100分）→300秒滑动窗口趋势检测→24小时自适应基线异常检测（均值±2σ）→控制引擎评估→终端ANSI多行刷新→CSV按日写入。

**模块2：双层决策控制引擎（control.py）**

6条优先级本地规则（P0 t>35°C风扇全速→P1 t>30°C比例散热/soil>2500浇水→P2 light<50补光/h<45%加湿→P3 soil<800停水）。LLM在满足事件触发条件时（土壤连续10次偏干、温度5分钟上升>3°C、健康指数<60持续1分钟）异步提供建议，经硬安全上限过滤后执行。连续失败3次自动暂停5分钟并降级为纯本地规则。

**模块3：情感映射引擎（agent/emotion.py + expression_engine.py）**

这是虚实融合的核心——建立从物理世界到虚拟角色的映射管线：

```
传感器数据 (t=32°C, h=45%, soil=2800...)
    │
    ▼
分析引擎 (health=62, trend: 温度↗+2.1°C/5min, alert: 土壤偏干)
    │
    ▼
LLM语义转换 ("土壤有点干了，温度也在上升，我感觉不太舒服...")
    │
    ▼
情绪分类器 (emotion="uncomfortable", intensity=0.7, reason="高温干旱")
    │
    ▼
VTube Studio 表情参数
  ├── EyeOpenLeft/Right 微微眯眼 (不适)
  ├── MouthSmile 下降 (不开心)
  ├── BrowLower 微蹙眉 (担忧)
  └── ParamHeadY 微微低头 (蔫蔫的)
```

8种情绪标签与传感器状态的对应关系通过LLM动态理解而非静态映射表，使得同一传感器数据在不同语境下可以产生不同的情绪表达。

**模块4：Agent AI大脑（agent/agent.py）**

LangGraph状态图工作流（记忆→RAG→搜索→回答），全部在ELF 2本地运行。角色人设系统（persona.py定义身份、expertise.py定义知识分级策略、conversation.py定义对话风格）组合为约3,000字的系统提示词。Agent支持注入实时传感器上下文，使角色回复具有环境感知能力。

**模块5：视频NPU推理流水线（video/）**

TCP:6001接收H.265视频流→PyAV软件解码→OpenCV BGR帧→YOLOv5 NPU推理→连续3次确认+30秒冷却去重→标注截图保存→MQTT事件推送。

**模块6：MQTT云端通信（mqtt/subscriber.py）**

ELF 2作为MQTT核心节点，TLS加密连接EMQX Cloud，15条指令路由全部在ELF 2本地处理——直接查询sensor_state字典或系统信息即可完成响应。

**模块7：本地语音对话（conversation.py + tts_local.py）**

UART语音输入→意图路由（16条中文语音指令）→不匹配则进入Agent对话→Piper TTS合成（16kHz PCM）→aplay音频输出→VTube Studio WebSocket口型同步。PC端离线时ELF 2可独立完成语音对话全流程。

**第四部分 总结**

**12. 可扩展之处**

（1）**多节点群控**：利用ELF 2的双路千兆以太网和MQTT主题命名空间，扩展为多温室集中管理系统——一个ELF 2节点接入多个K230感知终端，通过主题隔离实现数据分流。

（2）**NPU模型扩展**：RK3588的6TOPS NPU目前仅部署YOLOv5，算力余量可承载更多模型——植物病害分类、虫害检测、果实成熟度判断，实现更全面的AI视觉分析。

（3）**本地大模型部署**：在ELF 2上部署经过RK3588 NPU适配的轻量化LLM（如TinyLlama-1.1B、Qwen2-0.5B），实现完全离线的情感映射与对话能力，彻底摆脱对云端API的依赖。

（4）**情感映射的深度学习优化**：当前情感映射依赖LLM语义理解，可训练专用的轻量级"传感器→情绪"映射模型，部署在NPU上实现更低延迟的情绪变化（从秒级降至毫秒级），使角色的表情响应更加细腻自然。

（5）**Web管理界面**：利用ELF 2的Ubuntu生态，部署Flask/FastAPI + Vue.js构建Web管理后台，提供历史数据图表、实时视频预览、对话记录查看等功能。

本项目的研发是一次"从电路板到灵魂"的完整旅程。从最初调试I2C总线上的传感器地址，到最后让虚拟角色因为土壤湿度变化而流露出担忧的表情，我们经历了从嵌入式底层到AI应用层的完整技术栈。

**关于"虚实融合"的思考**。这个项目最让我们兴奋的不是某个单一技术的突破，而是"物理世界与虚拟角色之间那条连线的建立"。当看到酢酱酱因为真实的传感器读数（而不是预设脚本）而表现出开心或担忧时，我们意识到——这不仅仅是数据可视化的另一种形式，而是一种新型的人机关系。用户不再"查看"植物状态，而是"关心"一个角色的感受。这种情感连接让智慧农业从工具变成了伴侣。

## 一、ELF 2 部署环境要求

### 1.1 系统信息

```
平台:     飞凌 ELF 2 开发板
芯片:     Rockchip RK3588
系统:     ELF Desktop (Ubuntu) 22.04
内核:     Linux 5.10.209
Python:   3.10.12 (系统自带)
```

### 1.2 基础环境安装

```bash
# === 1. 更新包管理器 ===
sudo apt update && sudo apt upgrade -y

# === 2. 安装系统依赖 ===
sudo apt install -y \
    python3-pip python3-dev python3-venv \
    portaudio19-dev espeak-ng alsa-utils \
    libsndfile1 libopenblas-dev \
    ffmpeg libavcodec-dev libavformat-dev libavutil-dev \
    git curl wget

# === 3. 创建项目虚拟环境 ===
python3 -m venv ~/greenhouse-venv
source ~/greenhouse-venv/bin/activate

# === 4. 升级 pip ===
pip install --upgrade pip setuptools wheel
```

### 1.3 安装 Python 依赖

```bash
# 进入项目目录
cd ~/elf2-projects/elf2/

# 安装核心依赖
pip install -r requirements.txt

# 如果 RK3588 是 aarch64，手动安装 Piper TTS
# (requirements.txt 中已有条件判断，但可能需要手动确认)
pip install piper-tts

# 下载 Piper 中文语音模型
mkdir -p ~/.local/share/piper-tts/
wget -O ~/.local/share/piper-tts/zh_CN-huayan-medium.onnx \
    https://huggingface.co/rhasspy/piper-voices/resolve/main/zh/zh_CN/huayan/medium/zh_CN-huayan-medium.onnx
wget -O ~/.local/share/piper-tts/zh_CN-huayan-medium.onnx.json \
    https://huggingface.co/rhasspy/piper-voices/resolve/main/zh/zh_CN/huayan/medium/zh_CN-huayan-medium.onnx.json

# === YOLOv5 依赖 (需要 PyTorch) ===
# ELF2 RK3588 aarch64: 使用 Rockchip NPU 适配版 PyTorch
# 参考: https://github.com/rockchip-linux/rknn-toolkit2
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu
pip install ultralytics opencv-python av

# === Agent 依赖 ===
pip install langgraph langchain-chroma chromadb zhipuai python-docx requests beautifulsoup4

# === MQTT ===
pip install paho-mqtt

# === 数据库 ===
pip install pymysql
sudo apt install -y mysql-server
```

## 二、三端启动流程

### ⚡ 启动顺序（必须按此顺序！）

```
1. K230 先启动  →  传感器上线，等待 ELF2 连接
2. ELF2 再启动 →  接收传感器，分析+控制+Agent 就绪
3. PC 最后启动 →  连接 MQTT，启动虚拟角色和平台交互
```

## 四、常见问题排查

| 现象 | 可能原因 | 解决 |
|------|---------|------|
| ELF2 收不到传感器数据 | K230 WiFi 未连接 / IP 变化 | 检查 K230 终端 WiFi 状态；确认 ELF2 静态 IP 正确 |
| `[SENSOR] 解析失败` 频繁 | K230 JSON 格式异常 | 检查 sensor_loop.py 的 ujson.dumps 输出 |
| CCS811 一直显示"预热" | 传感器需要 20 分钟预热 | 正常现象，等待即可 |
| YOLO 报 `CUDA/GPU not available` | RK3588 NPU 需要 rknn 后端 | CPU 模式下推理仍可运行，速度较慢 |
| MQTT 连接失败 | TLS 证书路径不对 / 网络不通 | 确认 `emqxsl-ca.crt` 路径；`ping p8c59112.ala.cn-hangzhou.emqxsl.cn` |
| Piper TTS 报 `FileNotFoundError` | 语音模型未下载 | 参考 1.3 节手动下载 |
| `ModuleNotFoundError: langgraph` | 未安装 Agent 依赖 | `pip install -r requirements.txt` |
| VTube WebSocket 连接失败 | VTS 未开启 API 或端口错误 | VTS 设置中启用 "Start API"，默认端口 8001 |

## 五、一键启动脚本

### ELF2 端 (`~/start-greenhouse.sh`)

```bash
#!/bin/bash
# ELF2 智能温室系统一键启动

PROJECT_DIR="$HOME/elf2-projects"
VENV="$HOME/greenhouse-venv/bin/activate"

echo "========================================"
echo "  ELF2 智能温室系统启动中..."
echo "========================================"

source "$VENV"

# 检查 MySQL
if ! systemctl is-active --quiet mysql; then
    echo "[INIT] 启动 MySQL..."
    sudo systemctl start mysql
fi

# 检查网络
echo "[INIT] 检查网络连接..."
ping -c 1 -W 2 8.8.8.8 > /dev/null 2>&1
if [ $? -ne 0 ]; then
    echo "[WARN] 网络不可用, LLM/搜索功能将受限"
fi

# 启动子系统A (温室控制器)
echo "[START] 子系统A: 温室控制器"
cd "$PROJECT_DIR/sensor-listener/"
python -m sensor_listener --verbose &
PID_A=$!

sleep 2

# 启动子系统B (AI角色分身)
echo "[START] 子系统B: AI角色分身"
cd "$PROJECT_DIR/elf2/"
python elf2_main.py &
PID_B=$!

echo "========================================"
echo "  系统已启动!"
echo "  子系统A PID: $PID_A"
echo "  子系统B PID: $PID_B"
echo "  Ctrl+C 停止全部"
echo "========================================"

# 等待任意子进程退出
wait -n

# 清理
echo "[STOP] 正在停止所有子系统..."
kill $PID_A $PID_B 2>/dev/null
wait
echo "[STOP] 已停止"
```
